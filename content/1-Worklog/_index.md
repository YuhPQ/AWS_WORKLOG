Typically, and as a standard practice, a worklog is carried out over approximately 3 months (throughout the internship period) with weekly activities as follows:

**Week 1:** [Getting familiar with the internship environment and project orientation](1.1-week1/)

**Week 2:** [Learning basic AWS services and getting familiar with AWS Console and CLI](1.2-week2/)

**Week 3:** [Requirement analysis and system architecture design (MVC, technology orientation)](1.3-week3/)

**Week 4:** [Database design and researching system architecture (SQL vs NoSQL)](1.4-week4/)

**Week 5:** [Building basic backend (NodeJS, Express, Prisma)](1.5-week5/)

**Week 6:** [Developing authentication and user management system (Authentication & Authorization)](1.6-week6/)

**Week 7:** [Implementing cart functionality and business logic processing (Cart System)](1.7-week7/)

**Week 8:** [Completing checkout flow and integrating AWS S3 storage (Order & Checkout)](1.8-week8/)

**Week 9:** [Standardizing backend, implementing role-based access control and building admin APIs](1.9-week9/)

**Week 10:** [Building admin dashboard UI and integrating system modules](1.10-week10/)

**Week 11:** [Deploying the system to AWS (ECS, S3, CloudFront, CloudWatch)](1.11-week11/)

**Week 12:** [Testing, optimizing the system, and completing report & demo](1.12-week12/)
